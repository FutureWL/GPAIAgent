# 修复 `/api/auth/login` 500 错误

## Goal

定位并修复登录接口 500 错误的根因，使 `POST /api/auth/login` 正常返回。

## 当前错误

```
POST http://localhost:3000/api/auth/login 500 (Internal Server Error)
```

请求从 Next.js Web (`:3000`) 发出，Next.js 作为代理调用后端 API `:3001`（或 `:3002`）。

## 排查步骤

### Step 1：确认后端 API 是否可达

```bash
# 查看当前 API 运行端口
grep "PORT" apps/api/.env.dev

# 测试登录接口（绕过 Next.js 代理直接打后端）
curl -X POST http://localhost:3001/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"devtest","password":"devpass123"}' \
  -v 2>&1 | tail -20
```

### Step 2：确认 Next.js 代理路由

路径：`apps/web/src/app/api/auth/login/route.ts`

检查：
- 请求体解析是否正确（`request.json()` 是否在 try/catch 内）
- `NESTJS_BASE` fallback 是否为错误端口
- 后端返回 500 时 Next.js 是否直接透传

### Step 3：查看后端 auth controller 日志

路径：`apps/api/src/auth/auth.controller.ts`

检查 `login` 方法：
- 是否有未捕获的异常（数据库连接失败、bcrypt 错误等）
- `@Body()` 参数是否与请求 JSON 匹配
- JWT secret 是否存在

### Step 4：检查 auth.service.ts login 逻辑

路径：`apps/api/src/auth/auth.service.ts`

常见 500 原因：
- `bcrypt.compare` 失败（密码不匹配但内部异常）
- `jwtService.sign` 失败（JWT_SECRET 未设置或无效）
- Prisma 查询用户时数据库连接失败

## 关键文件

| 文件 | 作用 |
|------|------|
| `apps/web/src/app/api/auth/login/route.ts` | Next.js 代理入口 |
| `apps/api/src/auth/auth.controller.ts` | 后端 login 端点 |
| `apps/api/src/auth/auth.service.ts` | login 业务逻辑 |
| `apps/api/.env.dev` | 数据库/JWT 配置 |
| `apps/api/prisma/schema.prisma` | User 模型定义 |

## 验证步骤

1. 直接 curl 后端 auth 接口，确认是后端问题还是 Next.js 代理问题
2. 如果后端直接报错，日志会显示具体异常堆栈
3. 修复后重新 `curl -X POST http://localhost:3000/api/auth/login ...` 验证
