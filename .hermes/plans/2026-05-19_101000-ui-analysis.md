## 二、P0 详解

### P0-1：antd + shadcn 混用 — dark mode 样式割裂

**根因**：`antd` 组件（Avatar/Dropdown/Tag/Tooltip/Button）在 `header.tsx` 和 `sidebar.tsx` 中使用 antd 自带的 Token 系统（`theme.useToken()`），**不经过 CSS 变量**，因此切换 dark mode 时这些组件的颜色不会随 `globals.css` 里的 CSS 变量变化。

**现象**：dark 模式下 Header 的 Avatar 背景色、Dropdown 背景、Tag 颜色仍然保持亮色风格，与页面其他部分不协调。

**修复方案**：将 Header 和 Sidebar 的 antd 组件逐步迁移到 shadcn/ui 对应组件：

| antd 组件 | shadcn 替换 | 现状 |
|-----------|-------------|------|
| `antd Avatar` | `ui/avatar.tsx`（已有） | header/sidebar 均未使用 |
| `antd Dropdown` | `ui/dropdown-menu.tsx`（已有） | header/sidebar 均未使用 |
| `antd Tag` | `ui/badge.tsx`（已有） | sidebar 用，header 未用 |
| `antd Tooltip` | `ui/tooltip`（需 add） | 两处均用 antd Tooltip |
| `antd Button` | `ui/button.tsx`（已有） | header/sidebar 用 antd Button |

**受影响文件**：
- `apps/web/src/components/layout/header.tsx`
- `apps/web/src/components/layout/sidebar.tsx`

---

### P0-2：z-index 层次混乱

**当前 `appshell.tsx` 布局顺序**（从上到下）：

```
<div h-screen>                          ← bg-background
  <div relative z-20>  ← Sidebar        (左侧固定导航)
  <div flex-1 flex flex-col>
    <div relative z-10> ← StockTicker    (行情滚动，z-10)
    <div relative z-20> ← Header         (顶部栏，z-20)
    <main>             ← 内容区
```

**问题**：
1. StockTicker(z-10) < Header(z-20)：StockTicker 被 Header 底部遮盖，滚动时内容可能被 Header 截断
2. AppShell 注释说 "Sidebar z-20 层次高于顶部导航栏"，但 Sidebar 是 `relative` 独立容器，实际不与 Header 在同一 stacking context，没有层叠问题
3. Admin layout 用的是独立 full-screen flex，两套 z-index 系统无关联，但侧边栏的 z-20 和主站 AppShell 的 z-20 可能产生意外的 fixed 元素遮挡

**修复方案**：
- StockTicker 改为 `z-30`（在 Header 之上滚动）
- AppShell 注释与实际代码对齐，移除误导性注释
- Admin layout 的 sidebar 没有 fixed/sticky，不需要 z-index

---

### P0-3：Admin Toaster 重复挂载

**当前代码**：
```tsx
// admin/layout.tsx
<main className="...">
  <Providers>{children}</Providers>   // Toaster 在这里
</main>
```

**问题**：`Providers`（含 `<Toaster>`）在每次路由切换时重新渲染，导致 Toaster toast 闪现/抖动。

**修复方案**：
- 将 `Providers`（只含 Toaster）提升到 `layout.tsx` 的 JSX root 层，而不是作为 children 的包装
- 或在 `(market)` group layout 中统一提供 Providers

## 三、P1 详解

### P1-3 & P1-4：market page 股票硬编码 + 无防抖

`(market)/market/page.tsx` 一次性请求 70+ 支股票，每次 mount 都重新请求，没有缓存、没有分批、没有防抖。

**修复方案**：
- 股票列表从后端 `/stocks` 端点获取（已有 stocks service）
- 批量请求改用分批（每批 10-20 支，间隔请求）
- 添加 `useDeferredValue` 或 `useTransition` 防止阻塞 UI
- 添加本地缓存（React cache 或 SWR 模式）

---

**修复方案**：提取到 `apps/web/src/lib/stock-utils.ts`，统一导出。

## 四、P2 详解

### P2-1：Admin header 缺少主站体验

admin layout 的顶栏只有"管理控制台"文字 + 退出按钮，缺少：
- Logo（主站有 `TrendingUp` SVG）
- 语言切换（主站 Header 有）
- 主题切换（主站 Header 有）
- 面包屑或当前页面名称（没有）

**修复方案**：AdminHeader 组件复用主站 theme/language switcher 逻辑，或抽离出可复用的 HeaderRight 组件。

## 七、风险与权衡

1. **antd 组件完全移除后，某些复杂交互（如图标选择器、日期选择器）需要找 shadcn 替代或保留 antd**。建议只迁移纯展示类组件（Avatar、Tag、Badge、Tooltip），交互复杂组件（如 DatePicker）保留 antd。
2. **market page 批量请求改造可能影响现有实时行情功能**，需要充分测试。
3. **dark mode 迁移过程中可能出现短暂的样式闪烁**，建议分阶段提交。
