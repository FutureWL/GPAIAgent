# 个股详情页（/zh/stock/[code]）重新设计方案

**日期**: 2026-05-20
**目标**: 修复数据加载 + 重新设计专业金融终端风格布局

---

## 一、现状问题

### 根本原因：API 路由 404
- 前端 `/api/stocks/${code}/quote` → 调用 NestJS 后端
- NestJS 后端没有 ` StocksController`（只有 `MarketController`）
- 导致所有 API 请求返回 500，数据完全空白

### 布局问题
- 只有顶栏 + 返回按钮，主体区域全空白
- 数据加载失败后无任何降级展示

---

## 二、解决方案

### 数据层：直接调用腾讯行情 API
market page 已有完整实现：`fetchQuotesBatch()` 从 `qt.gtimg.cn` 获取数据，绕过不存在的 NestJS 路由。

**复用腾讯接口**：
```
https://qt.gtimg.cn/q=sz000001  → 单股实时行情
https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?_var=kline_dayqfq&param=sz000001,day,,,,,300,qfq  → 日K线
```

## 三、实施步骤

### Step 1: 重写数据层
- 移除所有 `/api/stocks/${code}/*` 调用
- 新增 `fetchTencentQuote(code)` 直接调用 `qt.gtimg.cn`
- 新增 `fetchTencentKline(code, period, days)` 直接调用 `ifzq.gtimg.cn`
- 保持 `parseTencentLine` 解析逻辑（已有，可复用）

### Step 2: 重组页面布局
- 顶部信息区：`name / code` 左侧，`price + change%` 右侧大字
- 附加信息行：`今开 / 最高 / 最低 / 成交量 / 成交额`
- 周期选择 Tab：`分时 | 5日 | 日K | 周K | 月K | 季K | 年K | 1min | 5min | 15min | 30min | 60min`
- 图表区：K线（主）+ 成交量（次），ECharts，主题色同步
- 底部指标：`2列4行` grid，左列基本信息，右列交易数据

### Step 3: 主题适配
- 所有颜色用 CSS 变量：`hsl(var(--stock-up))` / `hsl(var(--stock-down))`
- 涨跌色正确：A 股标准（涨红跌绿）
- ECharts 背景跟随主题

### Step 4: 响应式
- 桌面：左侧基本信息 + 右侧交易数据（2列）
- 移动：堆叠为单列
